# traqo v0.2.0 — Everything is a Span

## Context

traqo v0.1.0 has a dedicated `llm_call` event type that ties the library to LLM use cases. Following the OpenTelemetry model (and what opik/langfuse do), everything should just be a span with metadata. An LLM call, a DB query, an HTTP request — all spans with different `metadata` dicts. This makes the library general-purpose while still working great for LLM tracing.

Additionally, the README has competitor comparison tables that should be replaced with a clean value proposition.

## Breaking Changes (v0.1.0 → v0.2.0)

- `llm_call` event type removed — everything is `span_start`/`span_end`
- `tracer.llm_event()` removed
- `tracer.span()` yields a `Span` object instead of a string span_id
- `trace_end` stats: `llm_calls` field removed
- Child summary: `llm_calls` → `spans` + token totals
- 5 event types instead of 6

## Data Model (5 event types)

| Type | Purpose |
|---|---|
| `trace_start` | Session begins, metadata |
| `span_start` | Something begins — `input`, `metadata` |
| `span_end` | Something ends — `output`, `metadata`, `duration_s`, `status` |
| `event` | Point-in-time log (`tracer.log()`) |
| `trace_end` | Session ends, aggregate stats |

`metadata` on spans is the universal extension point. LLM data (`model`, `token_usage`, `provider`) lives there.

## New `Span` Class

Yielded by `tracer.span()`. Mutable handle for setting output and metadata during execution:

```python
class Span:
    id: str            # immutable
    name: str          # immutable
    parent_id: str     # immutable
    output: Any        # set during execution, written to span_end
    metadata: dict     # accumulated during execution, written to span_end
```

Methods: `set_output(value)`, `set_metadata(key, value)`, `update_metadata(dict)`

## File Changes

### 1. `traqo/tracer.py` (core changes)
- Add `Span` class (before Tracer, ~25 lines)
- Rewrite `span()`: accept `input` + `metadata`, yield `Span` object, write metadata to span_start/span_end, call `_accumulate_tokens()` on exit
- Add `_accumulate_tokens(span)`: if `span.metadata` has `token_usage`, sum into stats
- Remove `llm_event()` method entirely
- Remove `_stats_llm_calls` attribute
- Update `__exit__` trace_end stats: remove `llm_calls`
- Update `_write_child_ended()`: replace `llm_calls` with `spans`, `total_input_tokens`, `total_output_tokens`

### 2. `traqo/decorator.py` (major simplification)
- Add `metadata` parameter to `@trace()`
- Replace both sync_wrapper and async_wrapper with delegation to `tracer.span()`:
  ```python
  with tracer.span(span_name, input=input_data, metadata=metadata) as span:
      result = func(*args, **kwargs)
      if capture_output:
          span.set_output(serialize_output(result))
      return result
  ```
- Remove imports: `_get_parent_id`, `_now`, `_pop_span`, `_push_span`, `_uuid`, `serialize_error`
- ~192 lines → ~60 lines

### 3. `traqo/integrations/openai.py`
- Replace `tracer.llm_event(...)` with span wrapping:
  ```python
  with tracer.span("openai.chat.completions.create", input=messages_if_capture, metadata={"provider": "openai"}) as span:
      response = self._completions.create(**kwargs)
      span.set_metadata("model", model)
      span.set_metadata("token_usage", usage)
      if tracer._capture_content:
          span.set_output(text)
      return response
  ```
- Remove `time` import (span handles duration)
- Same pattern for sync and async
